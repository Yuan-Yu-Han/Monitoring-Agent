# 监控系统架构总结

这是一个融合 **YOLO 目标检测** 与 **Qwen3-VL 多模态 Agent** 的智能监控系统，采用 **事件驱动 + 对话驱动** 混合模式。

---

## 系统数据流

```
RTSP 视频流 → [YOLO 检测] → [事件触发器] → [Agent 分析]
  ↑                                           ↓
  └─────────← [交互式对话] ←────────────────┘
                (用户提问)
```

## 工作流程

### 1️⃣ 视频处理
```
RTSP 源 → 流媒体服务 → WebSocket → 前端显示 + 帧捕捉
```
- 实时显示监控画面
- 支持从前端截取帧进行检测

### 2️⃣ 目标检测
```
帧 → YOLO → 检测结果 {"class": "person", "confidence": 0.92}
```
- YOLOv8n 模型
- 识别 80+ 类别，支持置信度阈值

### 3️⃣ 事件驱动（智能升级）
```
检测结果 → [状态机]
  第1次: IDLE → SUSPECT
  第3次: SUSPECT → ALARM (触发 Agent)
  无检测: 回到 IDLE
```
- ✅ **避免频繁调用 LLM**：仅在状态转换时触发
- ✅ **多次确认**：需要 3+ 次同一类型检测才升级到告警

### 4️⃣ Agent 分析（事件驱动）
```
Event → [Agent 接口] → [Qwen3-VL] → Tool Call
                         ↓
                    调用工具：检测、绘图、生成报告
                         ↓
                      返回分析结果
```
- 支持工具：`detect_image`、`draw_bboxes`、`generate_report`、`list_images`
- 自动调用工具完成复杂分析
